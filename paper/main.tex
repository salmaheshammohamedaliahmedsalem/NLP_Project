\begin{abstract}
The rapid adoption of large language models has made it increasingly difficult to distinguish human-written text from machine-generated text. While many AI-text detectors report high accuracy under random train-test splits, such results may overestimate real-world reliability when datasets contain leakage, duplicate samples, generator-specific artifacts, or label-revealing metadata. This paper presents a leakage-aware and cross-dataset study of binary AI-generated text detection. The project first audits a legacy Global AI vs. Human Content dataset and shows that near-perfect within-dataset performance is not sufficient evidence of robust detection. The final experiments compare four approaches: word-level TF-IDF with Logistic Regression, character-level TF-IDF with Linear SVM, stylometric features with Random Forest, and a proposed hybrid model combining word TF-IDF, character TF-IDF, and stylometric machine-signature features. Experiments are conducted on SemEval-2024 Task 8 English data, a balanced RAID English subset, cross-dataset transfer settings, and an Arabic proof-of-concept dataset. The hybrid model achieves the best SemEval F1-score of 0.9443, while character-level TF-IDF achieves the best RAID F1-score of 0.9528. However, cross-dataset evaluation reveals a clear generalization gap, with best F1 dropping to 0.7887 for SemEval-to-RAID transfer and 0.5977 for RAID-to-SemEval transfer. The findings suggest that robust AI-text detection should be evaluated beyond single-dataset accuracy and should include leakage analysis, feature comparison, and external transfer testing.
\end{abstract}

\section{Introduction}
Large language models (LLMs) can now generate fluent and coherent text across academic, professional, social, and creative domains. This creates practical concerns for plagiarism detection, misinformation monitoring, authorship verification, and assessment integrity. Consequently, the task of distinguishing human-written text from AI-generated text has become an important binary classification problem in natural language processing.

However, AI-generated text detection is not a solved problem. A detector may achieve high accuracy on a random split of one dataset while failing on text generated by unseen models, unseen prompts, different domains, or adversarially edited outputs. Recent benchmark work such as RAID emphasizes that many detectors are insufficiently tested under robust out-of-domain and adversarial conditions \cite{dugan2024raid}. Similarly, SemEval-2024 Task 8 formalizes machine-generated text detection across multiple generators, domains, and languages, highlighting the need for standardized evaluation \cite{wang2024semeval}.

This project addresses the problem from a leakage-aware and robustness-focused perspective. Instead of treating the task as only a high-accuracy classifier, the work investigates whether different feature families capture different types of machine signatures. Word-level TF-IDF captures lexical patterns, character-level TF-IDF captures surface and punctuation patterns, stylometric features capture writing-style statistics, and the proposed hybrid model combines these complementary signals.

The project also includes an important dataset audit component. Initial experiments used a legacy Global AI vs. Human Content dataset. Although the models reached nearly perfect within-dataset scores, inspection showed that the dataset contained strong leakage risks, including duplicate texts, metadata columns, and label-revealing artifacts. Therefore, the old dataset is retained as a legacy comparison and cautionary case study, while the main English experiments use SemEval and RAID.

The main contributions of this work are:
\begin{itemize}
    \item A leakage-aware redesign of an AI-vs-human detection project, where the old dataset is treated as a legacy audit rather than the final benchmark.
    \item A four-way comparison of lexical, character-level, stylometric, and hybrid feature representations.
    \item A cross-dataset evaluation showing that strong within-dataset results do not necessarily transfer across benchmarks.
    \item An Arabic proof-of-concept demonstrating how the same feature-based pipeline can be extended to non-English text.
\end{itemize}

\section{Related Work}

\subsection{Machine-Generated Text Detection Benchmarks}
SemEval-2024 Task 8 is one of the most relevant benchmark tasks for this project. It focuses on multidomain, multimodel, and multilingual machine-generated text detection \cite{wang2024semeval}. Subtask A is directly aligned with this project because it is a binary classification task that determines whether a text is human-written or machine-generated. The task includes both an English monolingual track and a multilingual track, making it suitable for the English main experiment and the Arabic extension.

RAID is another highly relevant benchmark because it focuses specifically on robust evaluation of machine-generated text detectors \cite{dugan2024raid}. RAID contains generations from multiple models, domains, decoding strategies, and adversarial attacks. Its main contribution is showing that high reported detector accuracy can be misleading if evaluation does not include unseen models or adversarial variation. This aligns strongly with the motivation of the present project, where cross-dataset transfer is used to test generalization.

HC3 introduced a human-ChatGPT comparison corpus and evaluated several detection methods for distinguishing ChatGPT responses from human answers \cite{guo2023hc3}. HC3 is relevant because it shows how generated responses can be compared against human expert responses and how dataset design affects detector behavior.

\subsection{Feature-Based and Statistical Detection}
Early work on generated-text detection showed that statistical patterns can help distinguish machine-generated text from human text. GLTR introduced statistical visualization methods to help humans detect generated text based on token probability ranks and generation artifacts \cite{gehrmann2019gltr}. Although GLTR is not the same as TF-IDF, it supports the broader idea that generated text may leave measurable statistical traces.

DetectGPT proposed a zero-shot detection method based on the curvature of a language model's log probability function \cite{mitchell2023detectgpt}. Unlike supervised classifiers, DetectGPT does not require a separate labeled training dataset. However, it depends on access to model probabilities and perturbation models. In contrast, the present project focuses on lightweight supervised models that are easier to reproduce in a course setting and can be applied without direct access to the generator model.

Several SemEval-2024 systems also explored different approaches to machine-generated text detection. Genaios used a Transformer Encoder combined with token-level probabilistic features from multiple LLaMA-2 models and achieved strong monolingual performance \cite{sarvazyan2024genaios}. DUTh compared classic machine learning algorithms and LLM-based methods for SemEval-2024 Task 8 \cite{kyriakou2024duth}. These works show that both classical and neural approaches remain relevant, especially when comparisons are made under shared benchmark settings.

\subsection{Stylometry and Machine Signatures}
Stylometry studies measurable writing-style characteristics such as sentence length, lexical diversity, punctuation usage, repetition, and other structural signals. These features are useful in authorship attribution and can also support AI-generated text detection. In this project, stylometric features are used to examine whether machine-generated content exhibits detectable writing-style signatures beyond vocabulary alone. The approach is related to studies that combine linguistic, statistical, and model-based signals for generated-text detection \cite{mindner2023classification}. The main difference is that this project explicitly compares stylometry alone against lexical, character-level, and hybrid representations under both within-dataset and cross-dataset settings.

\section{Datasets}

\subsection{Legacy Global AI vs. Human Dataset}
The project originally started with the Global AI vs. Human Content dataset stored as \texttt{ai\_vs\_human\_content\_v2\_20000.csv}. In the current repository, this dataset is preserved as a previous-project or legacy comparison baseline, not as the main final benchmark.

The current preparation summary reports 20,000 rows before cleaning, zero missing text values, 11,758 duplicate texts removed, and 8,242 final rows. The remaining samples contain 8,192 English text entries and only 50 code entries. The final columns include not only \texttt{text} and \texttt{label}, but also metadata fields such as type, source, topic, word count, character count, AI model, language, complexity score, and whether the text is multiline code. Such metadata is useful for audit analysis but must not be used as model input.

Within-dataset results on this legacy dataset were nearly perfect: M1, M2, and M3 reached 1.0000 F1, while M4 reached 0.9994 F1. These scores are treated as a warning sign rather than the main achievement. In AI-text detection, near-perfect performance on a single cleaned legacy dataset may indicate residual dataset artifacts or limited generalization difficulty. Therefore, the final paper emphasizes SemEval, RAID, and cross-dataset transfer results.

\subsection{SemEval-2024 English Dataset}
SemEval-2024 Task 8 Subtask A is used as the main English benchmark. The cleaned SemEval dataset contains 124,552 rows, with 65,646 human-written samples and 58,906 AI-generated samples. Labels are mapped only from explicit binary human/AI values, and unknown labels raise an error during preprocessing. Duplicate texts are removed during preparation.

\subsection{RAID English Subset}
RAID is used as a second English benchmark and robustness-oriented dataset. The prepared RAID subset contains 3,532 balanced samples: 1,766 human-written and 1,766 AI-generated. The subset is restricted to the abstracts domain in the current experiment. RAID metadata such as generator, attack, domain, and decoding strategy is preserved for analysis only and is not used as model input.

\subsection{Arabic Proof-of-Concept Dataset}
The Arabic proof-of-concept dataset contains 36,523 samples. It is highly imbalanced, with 2,992 human-written samples and 33,531 AI-generated samples. The Arabic preprocessing applies light normalization by removing diacritics and tatweel, normalizing Alef variants, normalizing Alef Maqsura to Ya, and collapsing whitespace. Punctuation is preserved because it may contain stylometric information. Due to class imbalance, the Arabic results are reported as proof-of-concept rather than a final robust Arabic benchmark.

\subsection{Preprocessing and Leakage Control}
All models use text as the only prediction input. Metadata fields such as generator, source, domain, attack, decoding strategy, source file, dataset name, topic, and AI model are excluded from training. This is important because such columns may reveal the label directly or indirectly. Each dataset is cleaned separately and is not mixed with another dataset during preprocessing.

For English datasets, the pipeline removes missing values and duplicate texts, maps labels to binary human/AI values, and creates reproducible train/test splits. For Arabic, light normalization is applied while preserving punctuation and other surface-level signals for stylometric analysis.

\subsection{Feature Representations}
Four feature representations are compared.

\subsubsection{Word TF-IDF}
Word-level TF-IDF represents text using word unigrams and bigrams. It captures lexical choices and frequent word patterns. In this project, word TF-IDF uses up to 50,000 features, English stopword removal, minimum document frequency of 2, and sublinear term frequency scaling.

\subsubsection{Character TF-IDF}
Character-level TF-IDF represents text using character 3-grams to 5-grams. This representation captures subword forms, punctuation habits, spacing, suffixes, and surface-level writing patterns. Character n-grams are especially useful when the exact vocabulary differs across datasets but surface patterns remain informative.

\subsubsection{Stylometric Features}
Stylometric features are handcrafted numerical features describing writing style. The implemented feature set includes word count, character count, sentence count, average word length, average sentence length, punctuation count, comma count, period count, question count, exclamation count, uppercase ratio, digit ratio, lexical diversity, repetition ratio, and stopword ratio. These features are designed to capture machine signatures in writing behavior rather than topic content.

\subsubsection{Hybrid Representation}
The proposed hybrid representation combines word TF-IDF, character TF-IDF, and stylometric features. The motivation is that lexical features capture what is written, character features capture surface-level structure, and stylometric features capture how the text is written. The hybrid vector is passed to a Linear SVM classifier.

\section{Experimental Setup}

The experiments are divided into four settings.

\subsection{Within-Dataset Evaluation}
Each model is trained and tested on the same dataset using a stratified split. This setting measures how well the model performs when training and test data come from the same distribution.

\subsection{Cross-Dataset Evaluation}
To test generalization, models are trained on one English dataset and evaluated on the other without retraining. Two transfer directions are evaluated: SemEval-to-RAID and RAID-to-SemEval. This is the most important robustness experiment because it measures whether detectors generalize beyond the dataset they were trained on.

\subsection{Legacy Dataset Evaluation}
The legacy dataset is evaluated separately as an audit baseline. Since within-dataset scores are nearly perfect, these results are interpreted cautiously and are not used as the primary claim of the paper.

\subsection{Arabic Proof-of-Concept}
The Arabic experiment evaluates M2 and M4 only. These are the most relevant models for Arabic because character-level patterns and hybrid representations are less dependent on English-specific vocabulary than word-based English TF-IDF. Due to strong class imbalance, these results are treated as preliminary.

The evaluation metrics are accuracy, precision, recall, F1-score, and ROC-AUC. F1-score is emphasized because it balances precision and recall.

\section{Discussion}

\subsection{Why the Hybrid Model Helps on SemEval}
The hybrid model achieves the best SemEval performance because it combines complementary signals. Word TF-IDF captures lexical patterns, character TF-IDF captures surface-level structure, and stylometric features capture statistical writing behavior. The improvement over M2 on SemEval is modest but meaningful because it suggests that stylometric signals can add value when the dataset is large and diverse.

\subsection{Why Character TF-IDF Wins on RAID}
On RAID, M2 outperforms the hybrid model. This suggests that character-level patterns are especially informative for the current RAID subset. Since the RAID subset used here is balanced but concentrated in abstracts, character n-grams may capture domain-specific formatting, phrase endings, punctuation, and model-specific surface artifacts more directly than the hybrid representation.

\subsection{Stylometry Alone is Interpretable but Weaker}
M3 performs worse than TF-IDF-based models on both English datasets. This means stylometry alone is not enough for the strongest classification performance. However, it remains valuable because it provides interpretable evidence about writing style. In practical AI-detection systems, interpretability matters because users need more than a binary label; they need signals explaining why text appears machine-like.

\subsection{Cross-Dataset Transfer is the Strongest Finding}
The most important finding is the cross-dataset generalization gap. A model can perform above 0.94 F1 within SemEval or RAID but drop substantially when transferred to another dataset. This supports the argument that AI-text detection must be evaluated under stricter conditions than random splitting. It also agrees with RAID's broader claim that detectors can be fooled by distribution shifts, sampling strategies, adversarial attacks, and unseen generators \cite{dugan2024raid}.

\subsection{How to Interpret the Old Dataset Results}
The legacy dataset scores are nearly perfect, but they should not be presented as proof that the detector solves the problem. Instead, they motivate the leakage-aware redesign of the project. The old dataset still has value because it demonstrates why dataset auditing, duplicate removal, and metadata exclusion are essential in AI-generated text detection research.

\section{Limitations}
Several limitations remain. First, the RAID experiment uses a balanced subset rather than the full RAID benchmark. Future work should evaluate across more RAID domains, attacks, decoding strategies, and held-out generators. Second, the Arabic experiment is highly imbalanced and should be repeated on a more balanced Arabic benchmark. Third, the current study focuses on lightweight classical models rather than fine-tuned transformers. This choice improves reproducibility and interpretability, but neural models may achieve stronger benchmark scores. Fourth, the code samples in the legacy dataset are too few after cleaning, so the project should not claim strong code-domain generalization.

\section{Conclusion}
This paper presented a leakage-aware and cross-dataset study of AI-generated text detection. The project began with a legacy dataset whose near-perfect within-dataset scores revealed the danger of relying on a single benchmark. The final experiments compared four model families across SemEval, RAID, cross-dataset transfer, and Arabic proof-of-concept settings. The proposed hybrid lexical-stylometric model achieved the best SemEval F1-score of 0.9443, while character-level TF-IDF achieved the best RAID F1-score of 0.9528. Most importantly, cross-dataset transfer performance dropped substantially, showing that high within-dataset accuracy does not guarantee robust generalization. These findings support the need for leakage-aware preprocessing, external evaluation, and multilingual testing in AI-generated text detection.
