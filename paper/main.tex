\begin{abstract}
Detecting whether content was written by a human or generated by a large language model is often presented as a standard binary classification task. However, public AI-detection datasets can contain shortcuts that inflate reported performance, including explicit generation markers, duplicated templates, and metadata fields that reveal the target label. This paper presents a leakage-aware experimental pipeline for the Global AI vs. Human Content 2026 dataset. We audit metadata, remove explicit AI markers, de-duplicate normalized text, remove conflicting duplicates, and evaluate several baselines under random and prompt-grouped splits. The study compares word-level TF-IDF, character-level TF-IDF, linear SVM, stylometric features, hybrid TF-IDF plus stylometry, and frozen DistilRoBERTa embeddings where available. Results show that after strict cleaning, performance is far lower than the perfect scores produced by leaky evaluations, with the best local challenge score obtained by TF-IDF Logistic Regression. The main contribution is therefore not an inflated leaderboard result, but a reproducible protocol for scientifically defensible AI-content detection.
\end{abstract}

\section{Introduction}
Large language models such as GPT-4, Gemini, and Claude can produce fluent text that is difficult to distinguish from human-authored writing. This motivates binary AI-content detection systems for education, journalism, code review, and online trust. A robust detector should avoid relying on superficial dataset artifacts and should generalize across prompts, topics, and writing styles.

The project addresses the Global AI vs. Human Content 2026 task as a supervised binary classification problem. The postgrad focus is stylometric analysis: rather than only learning topic words, the detector should capture machine-signature patterns such as repetition, punctuation regularity, sentence-length smoothness, and lexical diversity.

Initial experiments with the raw dataset produced perfect or near-perfect scores. A leakage audit showed that these scores were scientifically invalid because several fields and text markers directly exposed the label. The final pipeline therefore emphasizes data validity before model complexity.

\section{Related Work}
Transformer encoders such as BERT~\cite{devlin2019bert}, based on the attention architecture~\cite{vaswani2017attention}, are strong baselines for text classification. Prior work on neural text authorship attribution shows that generated text can carry model-specific artifacts~\cite{uchendu2020authorship}. Statistical detectors such as GLTR~\cite{gehrmann2019gltr} study token likelihood patterns as evidence of machine generation. Broader analyses of language-model release risks also emphasize the difficulty of reliable detection under distribution shift~\cite{solaiman2019release,pan2024llmdetect}.

This work follows the same direction but adds a stronger dataset audit. Instead of assuming the benchmark is clean, the pipeline tests for explicit label markers, metadata proxies, duplicated content, and prompt leakage.

\section{Dataset and Leakage Audit}
The raw dataset contains 20,000 rows with columns including content, label, type, source, prompt, topic, AI model, language, word count, character count, complexity score, and multiline-code flags. A direct audit found three major leakage risks.

First, every AI row contained an explicit ``AI-generated'' marker in the raw content. Second, the \texttt{source} and \texttt{ai\_model} columns were perfectly correlated with the target label. Third, normalized duplicate and conflicting examples were common. These findings make a naive random split unsuitable for publication-quality claims.

Therefore, this paper does not claim reliable code-domain generalization from the cleaned local data.

\section{Methodology}
\subsection{Preprocessing}
Preprocessing is intentionally light. The pipeline removes explicit AI markers and normalizes text for de-duplication, but preserves punctuation, capitalization, line breaks, and token structure for stylometric analysis. Labels are encoded as human = 0 and AI = 1. Leaky metadata columns are never used as model inputs.

\subsection{Stylometric Features}
The stylometric extractor computes character count, word count, sentence count, line count, average and standard deviation of word length, average and standard deviation of sentence length, lexical diversity, hapax ratio, word entropy, punctuation ratios, uppercase ratio, whitespace ratio, quote count, code-like brace/operator counts, repeated word ratio, repeated bigram ratio, repeated trigram ratio, and line-length statistics.

\subsection{Models}
The experiments compare five local models:
\begin{itemize}
\item TF-IDF word n-grams with Logistic Regression.
\item TF-IDF character n-grams with Logistic Regression.
\item TF-IDF word n-grams with calibrated Linear SVM.
\item Stylometric-only Logistic Regression.
\item Hybrid TF-IDF plus stylometric Logistic Regression.
\end{itemize}

A frozen DistilRoBERTa embedding experiment is included when cached transformer weights are available. It encodes text with a pretrained transformer and trains a Logistic Regression classifier on mean-pooled embeddings, optionally concatenated with stylometric features.

\section{Experimental Design}
The main split is a stratified random split with 70\% training, 15\% validation, and 15\% testing. A stricter prompt-grouped split uses prompt groups so that prompt templates do not appear in both train and test data. A separate challenge set contains 20 manually curated examples with unseen topics, messy human prose, structured AI prose, short texts, long texts, and code/text cases.

Three sanity checks are used: a label-shuffle text model, a length-only model, and a prompt-only model. A scientifically valid setup should push these controls near chance.

\section{Discussion}
The central result is that leakage-aware evaluation changes the story of the project. Raw metadata and explicit markers can make the task appear solved, but strict cleaning reveals a much harder classification problem. The challenge set further shows that high validation scores are not sufficient evidence of real-world robustness.

The stylometric-only model performs near chance on the cleaned split, indicating that the remaining text is not easily separable by simple style features alone. The hybrid model improves over some baselines on the random split but does not dominate across all conditions. This prevents an unsupported claim that hybrid stylometry universally outperforms TF-IDF. The strongest challenge result comes from TF-IDF Word Logistic Regression, suggesting that lexical cues still matter in the manually curated test examples.

\section{Limitations}
The cleaned dataset loses the code domain because many code samples are duplicated or label-conflicting after normalization. The challenge set is useful for stress testing but is small. Full transformer fine-tuning was not treated as the main reported result because it requires careful compute budgeting and should be repeated across seeds before publication-level claims.

\section{Conclusion}
This paper presents a reproducible, leakage-aware pipeline for AI-vs-human content detection. The main contribution is methodological rigor: explicit marker removal, metadata leakage auditing, duplicate and conflict removal, grouped splitting, challenge testing, and sanity checks. The results show that perfect scores on the raw dataset are suspicious rather than impressive. Future work should use a cleaner multi-domain dataset with non-conflicting code samples and larger out-of-distribution challenge sets.
